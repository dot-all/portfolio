## Engineering Philosophy

1.  **Product-First Engineering:** I believe software exists to solve business problems. I prioritize understanding the "why" before the "how" to ensure every line of code delivers measurable value.
2.  **Architecture as a Strategy:** Code is a liability that must be managed. I design maintainable, decoupled systems that allow teams to iterate quickly without accumulating unmanageable technical debt.
3.  **Pragmatic Iteration:** I embrace "Perfect Imperfection"—shipping value early to gather real-world data and iterating based on user feedback rather than over-engineering in isolation.
4.  **Data-Driven Performance:** Performance tuning is grounded in metrics. I rely on profiling and execution analysis to drive optimizations that impact the bottom line.
